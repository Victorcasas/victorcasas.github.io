Mailcatcher and Vagrant

Jul 30 2014 development

If tying to run MailCatcher on a VM managed by Vagrant you get the following error:

No data received. Unable to load the webpage because the server sent no data...

To solve this issue, kill your proccess and run:

$ mailcatcher --ip=0.0.0.0

Now you can access MailCatcher on your VM host using your VM's IP and port 1080. This way, Vagrant port forwarding isn't required.
