Useful Gems

Aug 1 2014 development

When I start every Rails project I add the following gems that help me in the development:

Thin

Thin is a Tiny, fast & funny HTTP server.

I use Thin instead of Webrick to avoid the following messages in development log:

[2014-07-12 22:00:00] WARN Could not determine content-length of response body. Set content-length of the response or set Response#chunked = true [2012-07-12 22:00:00] WARN Could not determine content-length of response body. Set content-length of the response or set Response#chunked = true

Installation

To install, add this line to development group in your Gemfile:

gem 'thin', group: :development

Then, from the command line, run:

$ bundle

Go to Thin website for more information.

Quiet Assets

Quiet Assets turns off the Rails asset pipeline log. So a lot of messages are not shown in the development log like:

Started GET "/assets/application.js?body=1" for 127.0.0.1 at 2014-07-12... Served asset /application.js - 304 Not Modified (8ms)

It's pretty good to watch only important messages.

Installation

To install, add this line to development group in your Gemfile:

gem 'quiet_assets', group: :development

Then, from the command line, run:

$ bundle

Disable it temporarily

It's can be disabled temporarily is you need placing the following in you config/development.rb:

config.quiet_assets = false

To more information go to Quiet Assets website.

Better Errors

Better Errors replaces the standard Rails error page with real useful error page.

Installation

It's important you only put in your development group.

Add this to your Gemfile:

group :development do gem "better_errors" end Then run:

$ bundle

Now, if you would like to use advanced features (REPL, local/instance variable inspection, pretty stack frame names) add Binding Caller gem to your Gemfile:

gem "binding_of_caller"

To more information go to Better Errors website.
